  STEP 3 ‚ÄĒ SUPABASE SETUP                                                       
                                                                                
  Create the following tables in Supabase. Write the SQL migration file:        
                                                                                
  users                                                                         
  - id (uuid, primary key, references auth.users)                               
  - username (text, unique, not null)                                           
  - created_at (timestamp)                                                      
                                                                                
  voice_notes                                                                   
  - id (uuid, primary key)                                                      
  - user_id (uuid, references users.id)                                         
  - audio_url (text, not null)                                                  
  - duration (integer, seconds)                                                 
  - created_at (timestamp)                                                      
                                                                                
  reactions                                                                     
  - id (uuid, primary key)                                                      
  - voice_note_id (uuid, references voice_notes.id)                             
  - reactor_user_id (uuid, references users.id)                                 
  - emoji (text, not null)                                                      
  - created_at (timestamp)                                                      
                                                                                
  deliveries                                                                    
  - id (uuid, primary key)                                                      
  - voice_note_id (uuid, references voice_notes.id)                             
  - delivered_to_user_id (uuid, references users.id)                            
  - heard (boolean, default false)                                              
  - created_at (timestamp)                                                      
                                                                                
  Enable Row Level Security on all tables. Write RLS policies:                  
  - Users can only read/write their own user row                                
  - Anyone authenticated can read voice_notes                                   
  - Anyone authenticated can insert reactions, but only to notes delivered to   
  them                                                                          
  - Deliveries are only readable by the recipient                               
                                                                                
  Create a Supabase Storage bucket called "voice-notes" with public read        
  access.
